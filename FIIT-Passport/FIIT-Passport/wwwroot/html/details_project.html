Подробности проекта

Примеры заполнения:

Цель проекта

Популяризация нейросетей, создание нового продукта с этими нейросетями

Результат проекта (продукт)

Веб-сайт/telegram бот, в котором пользователь сможет сгенерировать комикс по собственным идеям.
Примерный образ результата:
1. Пользователь придумывает тему комикса, отправляет её в сервис.
2. Сервис с помощью Гигачат тезисно продумывает историю на тему пользователя, выдает их.
3. Пользователь может редактировать тезисы истории - редактировать/удалять/добавлять. После подтверждает и отправляет на дальнейшую генерацию истории.
4. Сервис с помощью Гигачат продумывает из тезисов более подробную историю.
5. Пользователь может редактировать истории - редактировать/удалять/добавлять. После подтверждает и отправляет на дальнейшую генерацию картинок.
6. Сервис генерирует картинки по истории с помощью Кандинского и автоматически подставляет нужный текст на картинки.
7. По итогу пользователь получает комикс из изображений с наложенным текстом.

Критерии приемки результата/продукта

1. Сервис успешно генерирует картинки на основе истории и автоматически добавляет соответствующий текст на изображения.
2. Пользователь может процесс генерации истории (изменять/удалять/добавлять)
3. Сгенерированные тезисы и история имеют логический смысл и соответствуют выбранной теме
4. Сервис выдерживает не менее 10 пользователей одновременно
5. Пользователь может успешно создать комикс на свою тему
Технологический стек выбирает команда

Цель проекта

Повышение функциональности сайта реальной образовательной организации за счет интегрированной системы опросов

Результат проекта (продукт)

Необходимо создать внутри сайта организации «Школа олимпиадной математики Hogmaths» (сайт http://hogmaths.ru/) систему для проведения тестирования учащихся или проведения опросов учащихся. Система должна включать в себя:
- Конструктор для создания теста или опроса администратором сайта либо определенной категорией пользователей
- Реализацию процесса прохождения теста обычными пользователями сайта
Инструменты для публикации тестов, сбора результатов, сохранения и загрузки ранее созданных тестов
Это второй подход в созданию конструктора тестов. В прошлый раз команда сконцентрировалась на подготовке сайта к интеграции с будущим конструктором. Эти наработки можно использовать, либо написать заново.
В любом случае в этот раз приоритетом проекта является разработка полнофункционального конструктора тестов, а только затем интеграция его в сайт.

Критерии приемки результата/продукта

Конструктор теста:
- Гибкость в инструментах создания и редактирования теста
- Наличие возможности для хранения и публикации нескольких тестов
- Должны быть доступны следующие возможности редактирования теста:
- Добавлять и удалять фреймы, редактировать их, сортировать в нужном порядке
- Редактировать текст вопроса: изменять шрифт, размер, цвет текста и т.п. (небольшой текстовый редактор)
- Вставлять в текст формулы LaTeX
- Прикреплять изображения к тексту вопросов
- Выбирать форму фрейма в зависимости от типа получаемого ответа на вопрос: ввод числа, ввод текста, выбор одного или нескольких вариантов ответа, вопрос без ответа (информационный фрейм) и т.п.
- Настраивать тест: время прохождения, доступ, описание теста, показ результата
- Сохранять тест, загружать для дальнейшего редактирования, создавать копию теста
Прохождение теста:
- Понятный, не перегруженный интерфейс для пользователя, проходящего тест
- Соответствие оформления тестирования с общим дизайном сайта
- Нужно предусмотреть, чтобы доступ к прохождению тестирования имели только зарегистрированные учащиеся (например, имеющиеся в базе поданных заявок через форму регистрации на сайте).
- Ответы участников тестирования или опроса должны быть сохранены в таблице (можно гугл-таблицы), к каждому тесту своя таблица.
- Помимо ответов пользователей на тест необходимо собирать следующую информацию: IP пользователя, дата прохождения, затраченное время.
- Пользователь должен иметь возможность:
- Выбирать необходимый тест из доступных
- При прохождении теста возвращаться к любому из фреймов
- Получать информацию о полученных баллах по окончании тестирования (если тест или опрос подразумевает это)
Код и развертывание:
- Функциональность интегрирована в кодовую базу сайта
- Функциональность развернута вместе с сайтом в Яндекс.Облаке и доступна
Функциональность доступна на сайте http://hogmaths.ru/

Цель проекта

Целью является решение проблемы отслеживания абонементов путем разработки онлайн расписания и системы записи на урок. Сервис позволит преподавателю быстро ориентироваться по прогрессу прохождения курса учениками, ученикам смотреть количество пройденных уроков, а новым клиентам быстро без лишних действий ознакамливаться с услугами и сразу записываться.

Результат проекта (продукт)

Сайт для ознакомления с услугами и с системой абонементов и отслеживания расписания и посещаемости каждого ученика. Доступ есть у администратора, личный кабинет ученика не предусмотрен.
Телеграм бот, который поддерживает администрирование, а также уведомляет подключившихся к нему учеников об окончании абонемента.
Критически важно протестировать функциональность или хотя бы ее часть на реальных учениках.
Опционально (если успеют):
- запись на индивидуальные занятие
- онлайн оплата и уведомление о состоянии оплаты
- генерация сертификатов для дальнейшей печати
- другие дополнительные фичи, которые будут предложены командой

Критерии приемки результата/продукта

- Разработаны прототипы интерфейсов сайта в Figma
- Корректное отображение абонементов, расписания, посещаемости на сайте
- Часть административных функций можно выполнять в телеграм боте
- Телеграм бот уведомляет об окончании абонемента
- Успешное прохождение юзабилити тестирования на текущих учениках

Цель проекта

Проект имеет две основные цели:
1. С одной стороны веб-сайт предлагает скидки пользователю, что позволяет пользователю снизить затраты;
2. С другой стороны, веб-сайт позволяет бизнесу (партнеру сайта) найти новых клиентов из числа пользователей

Результат проекта (продукт)

Конечным результатом проекта является создание веб-сайта с функциональными особенностями, включающими в себя личный кабинет пользователя и партнера, регистрацию пользователя и партнера, единую витрину предложений, возможность партнеру создать предложение для пользователя и т.д.
Ожидаемый результат этого семестра:
- Сайт с мобильной версией (но не мобильное приложение)
- Предоставление цифровых скидок к продуктам VK (например, VK Музыка), либо интеграция с каким-то другим партнером
- Личный кабинет студента с почтой университета
- Главная страничка с предложениями
Дальнейшие планы:
- Личный кабинет партнера: регистрация через ИНН с интеграцией через гос.сервисы. Далее — регион присутствия. Совсем далее — статистика и дашборды про применение скидок
- Админка для апрува/тех.поддержки
- Поддержка скидок как на цифровые товары, так и на физические
- Мобильное приложение

Критерии приемки результата/продукта

- Реализация функциональных требований является наивысшим критерием приемки результата
- Желательно использоваться современные, но проверенные технологии. Например, Python, React + TypeScript, PostgreSQL или YDB, Yandex Cloud
- Требуется передача прав заказчику
